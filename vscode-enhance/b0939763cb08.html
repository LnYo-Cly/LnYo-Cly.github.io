VScode+PicGo搭建图床

发表于 2021-07-22 更新于 2021-07-23 分类于 VScode调教 阅读次数：
本文字数： 741 阅读时长 ≈ 1 分钟

1. 下载PicGo插件

先在扩展商店中下载PicGo插件（别告诉我你连VS code都没有）

2. 选择搭建图床的网站

目前picgo只支持weibo, qiniu, tcyun, upyun, github, aliyun, imgur 和 smms这几种，不支持gitee

这里我选择用GitHub做演示

如果你用其他的网站，方法也都是差不多的

1. 先进入setting面板

2. 打开developer setting

3. 生成token


点击personal access tokens，再点击generate token


这里note可以随意 ，下面我们只需要选中repo就可以了


这里我们会得到一个token，我们记下它，因为刷新后我们就看不到了

到这里我们已经得到了最关键的token

3. 新建一个仓库，当作图床


先new一个仓库

这里输入仓库名 并选择public

生成就可以了，如果有需要可以新建一个子文件夹，把图片放到子文件夹里面（新建文件夹时输入一个“/”即 文件夹名+“/”就可以生成一个文件夹）

4. 配置PicGo


打开扩展设置，接下来什么都不用干，往下拉就行

到这里我们已经配置成功了

windows操作：

ctrl+alt+u 将剪切板的图片上传到github

ctrl+alt+e 选择本地文件上传

mac操作：

cmd+opt+u 将剪切板的图片上传到github

cmd+opt+e 选择本地文件上传

这些是成功上传的图片，不过国内这些图片可能会加载不出来，这里我们用cdn加速

我们替换设置中的custom url

https://cdn.jsdelivr.net/gh/用户名/仓库名

其他的什么都不需要改动

这样设置完之后我们在国内也可以很快的访问到图片了

图片上传成功后在vscode右下角会有这样的提示

如果没有，是不是哪一步错了？(づ￣ 3￣)づ
